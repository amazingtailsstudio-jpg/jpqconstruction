## 🚀 Quick Start

### Local Development

1. **Clone the repository:**
   ```bash
   git clone https://github.com/amazingtailsstudio-jpg/jpqconstruction.git
   cd jpqconstruction
   ```

2. **Run local server:**
   ```bash
   python3 -m http.server 8000
   ```

3. **Open in browser:**
   ```
   http://localhost:8000
   ```

### Add New Media

1. **Add images/videos to `/media/` folder:**
   - Supported formats: `.jpg`, `.png`, `.gif`, `.webp`, `.svg`, `.mp4`, `.webm`
   - Exclude logos and contact info (they're filtered out automatically)

2. **Regenerate manifest:**
   ```bash
   python3 generate_media_manifest.py --debug
   ```

3. **Commit and push:**
   ```bash
   git add .
   git commit -m "Add new project photos"
   git push origin main
   ```

4. **GitHub Pages updates automatically** (1-2 minutes)

## 📊 Media Management

### Supported Formats

**Images:**
- `.jpg`, `.jpeg` ✅ Universal
- `.png` ✅ Universal
- `.gif` ✅ Universal
- `.webp` ✅ Modern browsers
- `.svg` ✅ Vector format

**Videos:**
- `.mp4` ✅ Universal (recommended)
- `.webm` ✅ Chrome, Firefox, Edge

### Auto-Generated Manifest

The carousel automatically loads media from `media-manifest.json`, which is generated by the Python script:

```bash
python3 generate_media_manifest.py --debug
```

This script:
- Scans the `/media/` folder
- Filters out logos and contact info
- Generates JSON manifest
- Supports 12 image formats and 7 video formats

## 🐛 Troubleshooting

### Carousel not showing images?

1. Check browser console: `F12` → Console tab
2. Look for error messages (❌ symbols)
3. Verify `media-manifest.json` exists
4. Run: `python3 generate_media_manifest.py --debug`

### Images appear blank?

1. Ensure files are in `/media/` folder
2. Use supported formats (`.jpg`, `.png`, `.webp` recommended)
3. Convert `.heic` files to `.jpg` (Mac: Quick Actions → Convert Image)
4. Regenerate manifest

### Site takes time to deploy?

- GitHub Pages builds take 1-3 minutes
- Check Settings → Pages for status
- Refresh browser after 2-3 minutes
